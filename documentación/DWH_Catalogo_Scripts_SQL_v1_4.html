2. Detalle de Procesos de Ingesta (Extracción)

2.1. Procedimiento dw.sp_consolidar_staging()

Objetivo: Unificar el esquema dispar de múltiples orígenes en una tabla plana de staging.

  • Origen: RCOA (tmp_rcoa_bi), SUIA (tmp_coa_bi), JBPM (Sector, Categorías, Hidrocarburos).
  • Destino: stg.consolidado_proyectos
  • Lógica técnica: Ejecuta una serie de INSERT INTO masivos, mapeando campos heterogéneos a un esquema estándar del DWH para facilitar la carga de dimensiones.

3. Detalle de Procedimientos de Transformación (DW)

3.1. Procedimiento dw.sp_carga_dim_area()

Objetivo: Resolución de 111 brechas geográficas críticas mediante el motor de inferencia analítica.

  • Origen: stg.suia_areas_bi + ref.inec_dpa_2024.
  • Destino: dw.dim_area
  • Código Crítico:
CASE
  WHEN f.area_id = 1120 THEN 'NAPO' -- OFICINA TÉCNICA TENA
  WHEN f.area_id = 1081 THEN 'ESMERALDAS' -- DIRECCIÓN ZONAL 2
END as provincia

3.2. Procedimiento dw.sp_carga_hechos()

Objetivo: Cruce de todas las dimensiones y hechos de regularización ambiental.

  • Lógica de Integridad: Utiliza el patrón FK Guard. Mediante COALESCE(sk, 0), asegura que ningún registro quede fuera del DWH por falta de datos en origen, asignándolo automáticamente al registro maestro "No Definido".

4. Auditoría y Mantenimiento

4.1. Script de Integridad: setup_reference_data_v1_4.sql

Este script es el pilar de la resiliencia del DWH. Debe ejecutarse obligatoriamente antes de cualquier carga tras un proceso de limpieza.
