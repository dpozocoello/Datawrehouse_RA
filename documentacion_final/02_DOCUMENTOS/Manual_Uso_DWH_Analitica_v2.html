1. Seguridad y Accesos (Sección 8)

El esquema dw es de solo lectura (SELECT) para los usuarios de reporting (ej. dashboard_user). Los permisos de INSERT/UPDATE/DELETE están restringidos exclusivamente al usuario de servicio del ETL y al rol de administrador (postgres o equivalente). Los datos confidenciales como RUC de generadores están protegidos mediante vistas de seguridad si es requerido por el nivel de acceso.

2. Operación y Monitoreo (Sección 9)

Runbook: Solución al Error UniqueViolation

Si durante la ejecución el log arroja "llave duplicada viola restricción de unicidad", significa que una versión antigua de las restricciones sigue viva. Siga estos pasos:

1. Ejecutar script de emergencia para forzar la eliminación:
python D:\Datawrehouse_RA\scratch\force_cleanup.py

2. Re-aplicar el Procedimiento Almacenado y DDL:
python D:\Datawrehouse_RA\scratch\apply_sql_changes.py

3. Lanzar nuevamente el Orquestador:
python D:\Datawrehouse_RA\run_full_rgd_etl.py

3. Consumo y Dashboarding (Sección 10)

El consumo primario se realiza a través del módulo RGD (tab 9) en el Dashboard ECO-SIEAA Analytics.

  • Navegación Jerárquica: Las visualizaciones tipo Sunburst o Treemap ahora deben utilizar la ruta: Provincia -> Cantón -> Punto de Generación -> Tipo de Desecho gracias a la nueva dim_punto_generacion.
  • Filtros Temporales: Todo cruce debe hacerse utilizando la sk_tiempo, filtrando sobre la vista de fechas sincronizada para garantizar que los gráficos interactivos cambien en bloque.
